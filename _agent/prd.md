# Product Requirements Document (PRD)
## IPL Match Prediction Web App

**Version:** 1.0.0
**Date:** 2026-03-31
**Owner:** Jainil (Admin / Super User)

---

## 1. Overview

### 1.1 Product Vision
A private, role-based web application that displays IPL match predictions generated by a Claude AI skill (60-parameter framework). The admin uploads match prediction JSON, and registered viewers can access beautifully rendered prediction dashboards and historical accuracy reports.

### 1.2 Problem Statement
The Claude AI skill generates high-quality IPL match predictions, but the output is confined to the Claude interface. There is no persistent storage, no history tracking, no accuracy measurement, and no way to share predictions with others in a structured way.

### 1.3 Solution
A lightweight web app where:
- The admin (Jainil) uploads prediction JSON generated from Claude
- The system parses, stores, and renders it as a premium prediction UI
- Viewers (like Takshat) can log in and see current + historical predictions
- The system tracks prediction accuracy over time as actual results are entered

### 2.1 Role Definitions

**Admin (Super User)**
- Full access to all pages
- Can upload match prediction JSON files
- Can mark the actual result of a completed match
- Can add / deactivate viewer accounts

**Viewer**
- Read-only access
- Can see the latest prediction card
- Can see full history and accuracy table
- Cannot upload or modify any data

## 3. Core Features

### F1 — Authentication
- Login page with username + password
- JWT-based session (stored in localStorage / HttpOnly cookie)
- Logout from any page
- Auto-redirect to login if session expired
- Pre-seeded users: `jainil` (admin) and `takshat` (viewer)
- Passwords stored as bcrypt hashes

### F2 — JSON Upload (Admin Only)
- Drag-and-drop or file-picker to upload `.json` prediction file
- System validates JSON against the universal schema
- On success: stores the raw JSON + extracted fields in the database
- On failure: shows field-level error messages
- One active prediction per match ID (duplicate match_id is rejected)

### F3 — Prediction Dashboard (All Users)
- Displays the most recently uploaded match prediction
- Full premium UI with all sections:
  - Match hero banner
  - Winner card with confidence meter
  - 60-parameter dimension scorecard
  - Match context grid (venue, weather, H2H)
  - Playing XI side-by-side
  - Key player matchups
  - Top batters & bowlers stats
  - 3 reasons winner wins
  - Upset path conditions
  - Key risks & flip factors
  - Recent form W/L chips
  - Data limitations notice

### F4 — Match History (All Users)
- Table of all past predictions in reverse chronological order
- Columns: Match, Date, Predicted Winner, Confidence, Actual Result, Correct?
- Visual accuracy % meter at the top

### F5 — Mark Result (Admin Only)
- Per-match button: "Mark Actual Result"
- Admin selects the actual winning team
- System computes `is_correct` (true/false) and updates accuracy stats
- Once marked, result is locked (no re-editing)

### F6 — Accuracy Tracker
- Displayed prominently on the History page
- Shows: Total Predictions, Correct, Incorrect, Accuracy %
- Breaks down by confidence level: HIGH / MEDIUM / LOW accuracy rates

### F7 — Admin Panel (Admin Only)
- View all registered users
- Add new viewer accounts
- Deactivate users (soft delete)

### F8 — Real-time Chat Room (All Users)
- Instant messaging via WebSockets
- Single public room for match discussions
- Image upload and preview support
- Persistent history within the SQLite database
- JWT-protected secure communication

## 4. Non-Functional Requirements

| Requirement       | Specification                                      |
|-------------------|----------------------------------------------------|
| Performance       | Page load < 2 seconds on LAN                       |
| Security          | Passwords bcrypt-hashed; JWT auth; role enforcement |
| Portability       | Runs locally on Windows with `uvicorn`              |
| Database          | SQLite file (`data/ipl.db`) — no external DB server |
| Offline Support   | Works fully offline after first run                |
| Mobile Responsive | All pages usable on mobile screen widths            |
